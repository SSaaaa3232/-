# AI 第二大脑搭建指南

> 来源：X @rwayne
> 日期：2026-03-11

---

## 解决的3个痛点

| 痛点 | 问题 | 方案 |
|------|------|------|
| 失忆症 | 每次打开都要重新解释，AI不记得你是谁 | 三层记忆架构 |
| 同步地狱 | 云端越来越慢，离线不可用 | 本地化 + Git |
| 静态存储 | 不会自己整理、不会主动提醒 | 智能路由 + 任务追踪 |

---

## 12维度系统架构

### 1. 三层记忆架构

```
记忆库/
├── 情景记忆/     # 具体事件和对话历史（按月归档）
├── 语义记忆/     # 提炼的知识和模式
└── 强制规则/     # 系统行为约束
```

### 2. 智能存储路由

自动识别内容类型，自动决定存储位置：
- 文章类 → `01-内容/文章/`
- 笔记类 → `02-笔记/[主题]/`
- 项目文档 → `03-项目/[项目名]/`
- 工具脚本 → `04-工具/`

### 3. 任务追踪

你说"明天做XX"，AI自动添加到任务清单，并按时提醒。

### 4. 上下文监控

每次回复显示上下文使用率：
- 30% ⚠️ 开始提醒
- 50% 🟠 强烈建议
- 60% 🔴 必须清理

### 5. 外部知识吸收（6步法）

1. 深度阅读
2. 提取核心观点（3-5个）
3. 对比分析（我有什么/缺什么/能学什么）
4. 转化为行动（1-3个可执行改进点）
5. 记录到情景记忆
6. 提炼到语义记忆

### 6. 数据治理

**数据不删除，只转移**
- 任务归档 → `任务归档/YYYY/`
- 对话历史 → 每次会话追加

### 7. 苏格拉底式澄清协议（5Y）

决策/创作/方案类任务，必须先追问：
1. 你想解决什么问题？
2. 为什么这个问题重要？
3. 设想的结果是什么样的？
4. 有哪些约束或前提？
5. 确认理解，提出方案

### 8. Git 版本控制

所有改动可追溯、可回滚。
- 禁止 force push
- 禁止 git add -A

### 9. 自我迭代（8步）

1. 观察 - 回顾完整过程
2. 分析 - 什么做得好/可以改进
3. 设计 - 解决方案
4. 实施 - 更新文件
5. 验证 - 自我评分（<7分重做）
6. 记录 - 更新情景记忆
7. 提炼 - 更新语义记忆
8. 提交 - git commit

### 10. FATAL 铁律（宪法级）

| 规则 | 说明 |
|------|------|
| FATAL-001 | 禁止擅自重组文件结构 |
| FATAL-002 | 删除内容而非归档 |
| FATAL-003 | 忽略现有Obsidian链接 |
| FATAL-004 | 跳过结构性变更审批 |
| FATAL-005 | Git操作未经授权 |

### 11. 本地化

- 纯 Markdown
- 数据主权
- 离线可用
- 秒开

### 12. 完整目录结构

```
第二大脑/
├── CLAUDE.md              # AI总指南
├── 任务清单.md            # 唯一任务源
├── 对话历史.md            # 对话存档
├── 记忆库/
│   ├── 情景记忆/
│   ├── 语义记忆/
│   └── 强制规则/
├── 任务归档/
├── 日志/
├── 01-内容/
├── 02-笔记/
├── 03-项目/
└── 04-工具/
```

## 原文摘录

> 不是第二个Notion，是第二个大脑。
>
> 你用它，它就在进化。
>
> 数据在你手里，不在云端。

**核心理念：**
- 持久化记忆：不是对话工具，是长期伙伴
- 主动行动：不只是回答问题，会主动提醒、整理、进化
- 数据主权：所有数据本地存储，用户完全掌控
